# 串口通信可视化GUI程序 - 快速开始

## 快速开始（3步）

### 1. 确保依赖已安装

```bash
pip install pyserial
```

### 2. 选择版本运行

**基础版（推荐新手）**:
```bash
python radio_gui.py
```

**高级版（支持配置文件）**:
```bash
python radio_gui_advanced.py
```

**演示菜单**:
```bash
python demo_usage.py
```

### 3. 连接并使用

1. 输入串口号（默认 COM13）
2. 输入波特率（默认 38400）
3. 点击"连接"
4. 点击任意命令按钮发送数据
5. 在下方监听区查看接收数据

## 架构原理（简化版）

### 线程模型

```
主线程 (GUI)                监听线程 (后台)
    │                            │
    │                            │ 读取串口
    │                            ↓
    │                       [接收数据]
    │                            │
    │       Queue (线程安全)       │
    │ ←────────────────────────── │ put()
    │                            │
    │ 定时检查 (100ms)             │
    ↓                            │
  get()                          │
    │                            │
    ↓                            ↓
[更新GUI]                   [继续监听]
```

### 代码复用

- **protocol_parser.py**: 提供数据解析功能
  - `find_preamble()` - 查找协议头
  - `parse_frame()` - 解析数据帧

- **radio_simulator.py**: 提供数据发送功能
  - `send_xxx()` - 各种发送命令
  - `build_frame()` - 构建数据帧

- **线程安全**: 使用 `queue.Queue` 桥接两个线程

## 常见问题

### Q1: 串口打不开怎么办？

**A**: 检查以下几点：
- 端口号是否正确（Windows设备管理器查看）
- 是否被其他程序占用
- 是否有访问权限

### Q2: 为什么收不到数据？

**A**: 检查：
- 串口配置是否正确（波特率、停止位等）
- 外部设备是否正常工作
- 线缆连接是否正常
- 查看监听区是否有"监听线程已启动"消息

### Q3: GUI卡死了怎么办？

**A**: 这通常不会发生，因为程序使用了正确的线程设计。如果遇到，请：
- 强制关闭程序
- 检查是否修改了代码导致线程不安全
- 查看错误日志

### Q4: 如何添加新命令？

**A**:
1. 在 `radio_simulator.py` 中添加发送方法
2. 在 `radio_gui.py` 或 `radio_gui_advanced.py` 中添加对应的GUI方法
3. （高级版）更新配置文件添加按钮

### Q5: 数据解析失败怎么办？

**A**:
- 勾选"显示原始数据"查看HEX数据
- 检查PREAMBLE是否正确
- 查看 `protocol_parser.py` 的调试输出
- 确认数据格式符合协议规范
